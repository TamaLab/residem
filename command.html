Command Line options and arguments

residem

residem is a command line utility that computes isomorphous difference density (DED) maps. You will need to provide one reference PDB file along with an MTZ file of the reference and triggered system. The input can be intensity profiles or structure factor amplitudes, which the program will try to detect labels for. If labels for reflection files are not provided, they must be supplemented. The ResiDEM performs scaling of the reference and triggered structure factors, followed by the calculation of the difference structure factor and DED maps.

usage: residem [-h] --phil PHIL [-d DEFAULT] --ref REF --mtz MTZ --tmtz TMTZ
               [--high HIGH] [--low LOW] [--refl REFL] [--tril TRIL]
               [--fcalclabel FCALCLABEL] [--sigma SIGMA] [--weight WEIGHT]
               [--alpha ALPHA] [--scale SCALE] [--atom_profile ATOM_PROFILE]
               [--scaling_method SCALING_METHOD]
               [--ccp4_scaling_method CCP4_SCALING_METHOD]
-h, --help

show this help message and exit

--phil <phil>, -p <phil>

The PHIL file is a Python Hierarchical Input Language file, containing all inputs required to run the ResiDEM command line utility. Provide it with one argument.

-d <default>, --default <default>

Prints a default PHIL file which can be modified and provided as input.

--ref <ref>, -r <ref>

Reference PDB file used to generate phases and perform other calculations such as scaling, figure of merit, and phase error.

--mtz <mtz>, -m <mtz>

Refined reference MTZ file with experimental observed intensities or structure factors. It should also have FC and PHIC columns.

--tmtz <tmtz>, -t <tmtz>

Triggered MTZ file with experimental observed intensities or structure factors.

--high <high>, -hi <high>

High resolution cutoff. By default, the highest common resolution between the reference and triggered structure factors is used.

--low <low>, -l <low>

Low resolution cutoff. By default, the highest common resolution between the reference and triggered structure factors is used.

--refl <refl>, -rl <refl>

Label of the structure factor or intensity of the reference MTZ to be used for processing. By default, it identifies from labels such as ‘FP,SIGFP’, ‘F,SIGF’, ‘F-obs-filtered,SIGF-obs-filtered’, ‘I-obs,SIGI-obs’, or ‘I,SIGI’. If no label is identified, it must be provided.

--tril <tril>, -tl <tril>

Label of the structure factor or intensity of the triggered MTZ to be used for processing. By default, it identifies from labels such as ‘FP,SIGFP’, ‘F,SIGF’, ‘F-obs-filtered,SIGF-obs-filtered’, ‘I-obs,SIGI-obs’, or ‘I,SIGI’. If no label is identified, it must be provided.

--fcalclabel <fcalclabel>, -fl <fcalclabel>

Label of the calculated structure factor and phase of the reference MTZ to be used for processing. By default, it identifies from labels such as ‘FC,PHIC’, ‘FC_ALL,PHIC_ALL’, ‘FC,PHIFC’, or ‘F-model,PHIF-model’. If no label is identified, it must be provided.

--sigma <sigma>, -s <sigma>

Sigma cutoff for data processing. By default, it is 3.0 sigma.

--weight <weight>, -w <weight>

Weighting scheme for the treatment of noise in DED maps.

--alpha <alpha>, -ws <alpha>

Alpha for modified Ren weight. Use alpha=0.05 for Hekstra or alpha=1 for Schmidt weighting scheme.

--scale <scale>

Scaling of structure factors between the triggered and reference systems (derivative to native scaling) before generating difference structure factors and DED maps. It can be isotropic(iso), anisotropic(aniso), or no (no)scaling if the structure factors are already scaled.

--atom_profile <atom_profile>, -v <atom_profile>

One-dimensional atom-wise representation of difference density, based on max or mean peaks around the atom profile. By default, it is mean.

--scaling_method <scaling_method>, -sm <scaling_method>

Scaling method using methods implemented in CCTBX or CCP4-based programs.

--ccp4_scaling_method <ccp4_scaling_method>, -cs <ccp4_scaling_method>

If the scaling method selected is based on CCP4 programs, linear scaling can be chosen by default.

residem_coot

residem_coot is a command line utility used to visualize the DED maps. It uses Coot with an additional script to view residues associated with DED maps. It calls Coot and provides an input script for residual tracking associated with DED maps.

usage: residem_coot [-h] --ref REF --map MAP
-h, --help

show this help message and exit

--ref <ref>, -r <ref>

Reference PDB for model visualization

--map <map>, -m <map>

Isomorphous difference density (DED) map for visualization

residem_extract_around_model

residem_extract_around_model is a command line utility used to extract maps around a specified residue in the map.

usage: residem_extract_around_model [-h] --pdb PDB --map MAP --selection
                                    SELECTION [--radius RADIUS] [--box BOX]
                                    [--output OUTPUT]
-h, --help

show this help message and exit

--pdb <pdb>, -r <pdb>

Reference PDB for mask calculation

--map <map>, -m <map>

Isomorphous difference density (DED) map for masking in CCP4 format

--selection <selection>, -s <selection>

Residue selection for masking the DED map. Phenix/CCTBX atom naming should be given as input. Example: ‘resseq 182:214’

--radius <radius>, -d <radius>

Radius around the residue where the map should be truncated. Default is 5 Å.

--box <box>, -b <box>

Box cushion around the residue where the map should be truncated. Default is 5 Å.

--output <output>, -o <output>

Output name of the truncated map.

residem_svd_csv

residem_svd_csv is a command line utility used to analyze the results of difference density output. The utility can generate one-dimensional residue-wise plots, two-dimensional heat map plots, Pearson correlation plots, and network plots for residues with difference density in DED maps.

usage: residem_svd_csv [-h] --file FILE --cmap CMAP --ncmap NCMAP
                       [--closeness CLOSENESS] [--cutoff CUTOFF]
                       [--residue RESIDUE] [--pdb PDB] [--dist DIST]
                       [--low LOW] [--medium MEDIUM] [--map_type MAP_TYPE]
                       [--norm NORM] [--ew EW]
-h, --help

show this help message and exit

--file <file>, -f <file>

Files for analysis. List of files can be given, or if all files are in the same folder, “*.csv” can be provided as input.

--cmap <cmap>, -c <cmap>

Matplotlib colormap for plotting. Default is ‘Spectral’. More can be found at https://matplotlib.org/stable/users/explain/colors/colormaps.html

--ncmap <ncmap>, -s <ncmap>

Network colormap for plotting. Default is ‘Blues’ but can be changed to any sequential colors from https://matplotlib.org/stable/users/explain/colors/colormaps.html.

--closeness <closeness>, -k <closeness>

Subplot node closeness. Default is 0.05, but can be adjusted depending on the layout.

--cutoff <cutoff>, -i <cutoff>

Cutoff score for the data frame. The script either uses ‘electron_sum_around_the_site’ or ‘peak_sum’ from the input CSV. A cutoff score can be provided to mask values above it. Default is zero.

--residue <residue>, -n <residue>

Residues for analysis and plotting network. Example: ‘182,216-218,300’

--pdb <pdb>, -r <pdb>

Input reference PDB from which distance is calculated for network representation

--dist <dist>, -d <dist>

Distance calculation type. Available options are CA (C-alpha), CM (center of mass for non-hydrogen atoms), or SC (center of mass for non-hydrogen side chain atoms).

--low <low>, -l <low>

Low electron density sum cutoff threshold for plotting node size. Default is 0.05.

--medium <medium>, -m <medium>

Medium electron density sum cutoff threshold for plotting node size.

--map_type <map_type>, -t <map_type>

Type of map input. Options are ‘peak’ or ‘density’. By default, ‘density’ is chosen.

--norm <norm>, -z <norm>

Whether to normalize the input values. Must be either True or False.

--ew <ew>

Edge width between nodes. Default is 0.150.

residem_pymol_image

residem_pymol_image is a command line utility used to generate images of residues in DED maps using PyMOL.

usage: residem_pymol_image [-h] --pdb PDB --map MAP --resi RESI [--all ALL]
                           [--positive POSITIVE] [--negative NEGATIVE]
                           [--sigma SIGMA] [--carve CARVE] [--output OUTPUT]
                           [--rot_x ROT_X] [--rot_y ROT_Y] [--rot_z ROT_Z]
-h, --help

show this help message and exit

--pdb <pdb>, -r <pdb>

PDB file for analysis. List of files can be given, or if all files are in the same folder, “*.csv” can be provided as input.

--map <map>, -m <map>

Isomorphous difference density map in CCP4 format.

--resi <resi>, -resi <resi>

List of residues for which map and image should be generated, using PyMOL atom selection syntax. Example: ‘resi 182+216+300’. More details about [atom selection syntax](https://pymolwiki.org/index.php/Selection_Algebra)

--all <all>, -all <all>

If true, images of both positive and negative DED maps will be generated.

--positive <positive>, -p <positive>

If true, only an image of the positive DED map will be generated.

--negative <negative>, -n <negative>

If true, only an image of the negative DED map will be generated.

--sigma <sigma>, -s <sigma>

Sigma cutoff of the DED map. Default is +/- 3.0 sigma.

--carve <carve>, -c <carve>

Carve radius around the atom. Default is 1.2 Å.

--output <output>, -o <output>

Output name of the PNG file.

--rot_x <rot_x>, -x <rot_x>

Rotation around the x-axis in degrees.

--rot_y <rot_y>, -y <rot_y>

Rotation around the y-axis in degrees.

--rot_z <rot_z>, -z <rot_z>

Rotation around the z-axis in degrees.
